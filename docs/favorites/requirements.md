# Favourites Requirements

## Functional Requirements
1. Users can add or remove roasters from favourites.
2. Favourites are persisted in localStorage for guests and in the backend for authenticated users.
3. Users can view all their favourites on a dedicated page.
4. Favourites sync with the backend upon login.
5. The UI updates in real time when favourites change.

## Non-Functional Requirements
- Favourites must persist across sessions.
- API must validate authentication and roaster existence.
- UI must be responsive and accessible.

## User Stories
- As a user, I want to save my favourite roasters so I can find them easily later.
- As a user, I want my favourites to sync when I log in.
- As a guest, I want my favourites to persist in my browser.
